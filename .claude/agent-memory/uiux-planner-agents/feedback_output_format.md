description: 컴포넌트 명세 작성 시 테이블·와이어프레임 금지, 리스트+텍스트 전용으로 확정된 규칙
type: feedback
---

컴포넌트 명세는 테이블과 와이어프레임을 사용하지 않고, 리스트와 텍스트로만 작성한다.

**Why:** 사용자가 v1(ASCII 와이어프레임), v2(HTML 테이블 와이어프레임) 두 차례 모두 "원하는 아웃풋이 아니다"라고 피드백. 이유: 테이블은 Figma 렌더링 시 텍스트 잘림 문제 발생.

**How to apply:**
- 영역 구성(Section 2): `- ① **영역명** — 설명 / 필수 여부` 번호 리스트 형식만 사용. 모드가 여러 개면 소제목으로 분리
- Variant 목록(Section 3): 축별 소제목 + 글머리 기호 리스트
- 상태 정의(Section 4): 글머리 기호 리스트 (`- 상태명 — 시각적 변화 / 기능적 변화`)
- 이벤트 목록(Section 6): 글머리 기호 리스트
- 접근성 요구사항(Section 7): 글머리 기호 리스트
- 마크다운 테이블(`|---|`) 절대 사용 금지
- ASCII, HTML 와이어프레임 절대 사용 금지
