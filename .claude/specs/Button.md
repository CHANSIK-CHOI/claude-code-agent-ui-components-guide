# Button — 컴포넌트 기획

사용자 액션을 트리거하는 기본 인터랙션 컴포넌트

---

## 01 컴포넌트 개요

사용자가 특정 액션을 실행하도록 유도하는 인터랙티브 요소입니다. 폼 제출, 페이지 이동, 모달 열기, 데이터 저장 등 명시적인 사용자 의도를 시스템에 전달하는 가장 기본적인 액션 컴포넌트입니다.

주요 사용 맥락

- CTA (Call To Action)
- 폼 제출
- 다이얼로그 확인·취소
- 툴바 액션

---

## 02 영역 구성 (Area Map)

기본 모드 (label 포함)

- ① leftIcon — 레이블 왼쪽에 위치하는 보조 아이콘 / 선택
- ② label — 버튼의 텍스트 레이블, 액션 목적을 명시 / 필수
- ③ rightIcon — 레이블 오른쪽 보조 아이콘 (외부 링크, 방향 지시 등) / 선택
- ④ spinner — 로딩 상태일 때만 표시, leftIcon 위치를 대체 / 조건부

iconOnly 모드

- ① icon — 버튼의 의미를 단독으로 전달하는 중앙 아이콘 / 필수
- ② spinner — 로딩 상태일 때 icon을 대체하여 표시 / 조건부
- ③ accessibleLabel — 시각적으로 숨겨지지만 보조기기가 읽는 대체 텍스트 / 필수

---

## 03 Variant 목록

type — 시각적 위계

- primary — 화면에서 가장 중요한 액션. 폼 제출, 주요 CTA, 다음 단계 진행
- secondary — 보조 액션. 취소, 뒤로 가기, 대체 선택지
- outline — 테두리선 중심의 보조 CTA. 배경 없이 브랜드 색상 윤곽선으로 액션을 강조. 선택 강조·토글형 액션, secondary보다 가벼운 시각 위계가 필요한 보조 영역
- ghost — 배경 없는 경량 액션. 툴바, 테이블 행 내부, 아이콘 중심 영역
- danger — 파괴적·비가역적 액션. 삭제, 계정 해지, 데이터 초기화

size

- sm — 밀도 높은 영역. 테이블 셀, 툴바, 태그 옆 보조 액션
- md — 기본 크기. 일반적인 폼, 카드, 섹션 하단 액션
- lg — 강조가 필요한 화면. 온보딩 CTA, 랜딩 페이지, 주요 전환 지점

mode

- default (label 모드) — leftIcon / label / rightIcon 조합
- iconOnly — 아이콘 하나만 표시, 대체 텍스트 필수

---

## 04 상태 (State) 정의

- default — 기본 표시 상태. 시각적 변화 없음
- hover — 포인터 진입 시 배경이 한 단계 어둡거나 진해짐. 기능적 변화 없음
- focus — 키보드 포커스 진입 시 외곽선이 뚜렷하게 표시됨. 키보드 포커스 가능 상태
- active (pressed) — 클릭·탭 순간 배경이 hover보다 더 어둡거나 눌린 느낌으로 변화
- disabled — 색상이 흐려지고 커서가 차단 표시로 변경. 클릭 이벤트 차단, 포커스 건너뜀
- loading — spinner 표시, 레이블 또는 아이콘 대체. 클릭 차단, 버튼 폭 유지
- error (선택적) — 에러 색상으로 일시 표시. 실패 알림 후 default로 복귀

상태 조합 규칙

- disabled + loading 동시 적용 가능 → loading 시각 우선
- hover + focus 동시 적용 가능 → focus ring이 hover 배경 위에 겹침
- loading 중에는 hover / active 시각 변화 억제

---

## 05 동작 규칙

- loading 상태에서는 클릭 이벤트가 차단되며 중복 제출을 방지한다
- disabled 상태에서는 클릭 및 포커스 이동이 모두 차단된다
- disabled + loading 동시 적용 시 loading 시각 표현이 우선된다
- iconOnly 모드에서는 대체 텍스트가 반드시 제공되어야 한다
- 버튼 폭은 기본적으로 내부 콘텐츠 크기에 맞추며, 부모 컨테이너가 full-width를 제어한다
- loading 진입 시 버튼 폭과 높이는 유지되어 레이아웃 이동(CLS)이 발생하지 않는다
- leftIcon과 spinner는 동시에 표시될 수 없으며, loading 활성 시 spinner가 leftIcon 위치를 점유한다
- danger type은 확인 단계(모달·이중 확인)와 함께 사용하는 것을 원칙으로 한다
- 폼 내부에서 기본 submit 동작을 가질 수 있으며, 의도하지 않은 제출을 막기 위해 타입을 명시한다

---

## 06 이벤트 목록

- onClick — 버튼 클릭·탭 또는 포커스 상태에서 Enter·Space 눌렀을 때 발생. disabled·loading 상태 제외. 이벤트 객체 전달
- onFocus — 버튼이 키보드 또는 프로그램적으로 포커스를 받았을 때 발생
- onBlur — 버튼이 포커스를 잃었을 때 발생
- onMouseEnter / onMouseLeave — 포인터가 버튼 영역에 진입·이탈했을 때 발생. 툴팁 연동 시 활용

이벤트 차단 규칙

- disabled → 클릭·키보드 활성화 차단
- loading → onClick만 차단, 포커스·hover 피드백은 유지 가능

---

## 07 접근성 요구사항

- 키보드 접근 — 항상 포커스 이동이 가능해야 하며, Enter 또는 Space 키로 클릭 이벤트 실행 가능
- 대체 텍스트 — iconOnly 모드에서 aria-label 형태의 대체 텍스트 반드시 제공
- 비활성 상태 알림 — disabled 상태에서 보조기기가 '비활성'임을 사용자에게 전달
- 로딩 상태 알림 — loading 상태에서 보조기기가 '진행 중'임을 인지할 수 있도록 상태 변화 알림
- 포커스 표시 — 키보드 포커스 시 외곽선이 배경과 충분한 대비로 명확히 보여야 하며, 마우스 포커스와 구분
- 색상 대비 — 레이블과 배경의 대비는 WCAG AA 기준(4.5:1 이상) 충족
- 터치 영역 — 모바일 환경에서 최소 터치 영역 확보, sm size는 부모 padding으로 보완
- 역할 명시 — 링크 역할로 사용되는 경우 버튼이 아닌 링크 요소 기반으로 구현

---

## 08 구현 복잡도 신호

✅ 라이브러리 별도 검토 불필요 — 단일 트리거 컴포넌트로 직접 구현 권장

구현 시 주의 포인트

- loading 전환 시 레이아웃 흔들림(jitter) 방지 — spinner와 leftIcon 점유 크기를 동일하게 유지
- iconOnly 모드의 최소 터치 타깃 확보 — sm size에서도 접근성 기준 터치 영역 만족
- 폼 내부 배치 시 submit / reset / button 역할을 호출 측에서 명시적으로 지정 가능해야 함
